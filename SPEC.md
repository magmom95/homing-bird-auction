# 모의 부동산 경매 예측 이벤트 웹페이지 설계 문서

---

## 1. 개요

본 문서는 모의 부동산 경매 예측 이벤트 웹페이지의 설계 내용을 정의한다.

본 서비스는 실제 부동산 경매 형식을 기반으로, 사용자가 특정 경매 물건의 낙찰가를 예측하여 제출하고, 실제 낙찰가와 가장 근접한 사용자를 선정하는 이벤트형 웹페이지이다.

서비스는 회원가입 없이 이메일 기반으로 참여할 수 있으며, 경매 공고문 형태의 UI를 통해 몰입감을 제공한다.

---

## 2. 서비스 목적

* 스터디 및 지인 기반 참여형 이벤트 제공
* 부동산 경매 구조 이해 및 체험
* 간단한 예측 게임 형태의 콘텐츠 제공
* 최소한의 입력으로 높은 참여율 확보

---

## 3. 핵심 컨셉

* 봉투형 예측 (Sealed Prediction)
* 실시간 경쟁 없음
* 제출 금액 비공개
* 마감 후 결과 공개
* 당일 한정 결과 공개
* 1등 강조형 UX (모달)

---

## 4. 사용자 유형

### 4.1 일반 사용자

* 경매 물건 정보 확인
* 이메일 입력
* 예상 낙찰가 입력
* 예측 제출
* 결과 확인

---

### 4.2 관리자

* 관리자 페이지 접근
* 경매 물건 등록 및 수정
* 사진 업로드
* 감정가 / 최저입찰가 입력
* 마감일시 설정
* 결과 공개 제어
* 실제 낙찰가 입력

---

## 5. 서비스 구조

본 서비스는 두 개의 주요 페이지로 구성된다.

### 5.1 사용자 페이지

* 경매 공고 확인
* 예측 참여
* 결과 확인

### 5.2 관리자 페이지

* 경매 등록 및 관리
* 결과 입력 및 공개

---

## 6. 사용자 페이지 설계

---

### 6.1 화면 구성

#### 공고 영역

* 사건번호
* 물건명
* 주소
* 대표 이미지
* 감정가
* 최저입찰가
* 매수신청보증금 (자동 계산, 10%)
* 입찰 마감일시
* 현재 상태 (진행중 / 마감)
* 참고 링크 버튼

---

### 6.2 화면 예시

```text
사건번호: 2024-001

[대표 사진]

서울 강남구 ○○아파트

감정가: 12억
최저입찰가: 8.4억
매수신청보증금: 8,400만원 (10%)

입찰 마감: 2026-03-30 18:00
현재 상태: 진행 중

[ 자세히 보기 ]
```

---

### 6.3 입력 영역

```text
이메일
[ 입력 ]

닉네임 (선택)
[ 입력 ]

예상 낙찰가
[ 입력 ]

☑ 제출 후 수정이 불가함에 동의합니다.

[ 제출 ]
```

---

### 6.4 입력 규칙

* 이메일: 필수 입력
* 닉네임: 선택 입력
* 금액: 필수 입력
* 1인 1회 참여 제한 가능
* 제출 후 수정 불가

---

## 7. 보증금 개념

* 매수신청보증금은 최저입찰가 기준 10%로 자동 계산
* 사용자 입력 항목에는 포함하지 않음
* 안내 정보로만 표시

---

## 8. 입찰(예측) 흐름

```text
물건 확인
→ 이메일 입력
→ 금액 입력
→ 제출
→ 완료 화면
```

---

## 9. 결과 공개 구조

---

### 9.1 상태 정의

```ts
status: "open" | "closed"
resultOpen: boolean
```

---

### 9.2 공개 방식

* 결과는 **특정 날짜 (당일)에만 공개**
* 이후 자동 비공개

---

## 10. 결과 화면

---

### 10.1 기본 결과 화면

```text
실제 낙찰가: 9억 5천

🏆 TOP 3

1위 홍길동 (차이 100만원)
2위 김철수 (차이 200만원)
3위 박민수 (차이 350만원)
```

---

### 10.2 결과 표시 규칙

* 차이값 기준 오름차순 정렬
* TOP 3만 노출
* 전체 참여자는 표시하지 않음

---

## 11. 1등 모달 (핵심 UX)

---

### 11.1 모달 내용

```text
🎉 축하합니다!

이번 경매 예측 1위

홍길동

예상가: 9억 4,900
실제: 9억 5,000
차이: 100만원

[ 확인 ]
```

---

### 11.2 모달 조건

* 결과 공개 상태일 때
* 페이지 최초 진입 시 1회 노출

---

### 11.3 버튼

* 단일 버튼: 확인
* 클릭 시 모달 종료

---

## 12. 결과 공개 제한

```text
※ 결과는 당일에만 확인 가능합니다.
```

* 이벤트성 강화
* 긴장감 유지

## 13. 관리자 페이지 설계

---

### 13.1 관리자 진입

* 비밀번호 기반 접근

### 13.2 관리자 입력 항목

```text
사건번호
물건명
주소

대표 사진 업로드

감정가
최저입찰가

마감일시

참고 링크

결과 공개 여부

실제 낙찰가 입력

[ 저장 ]
```

## 14. 관리자 기능

## 15. 데이터 구조

---

### 15.1 경매 물건

```ts
type AuctionItem = {
  id: string;
  caseNumber: string;
  title: string;
  address: string;
  imageUrl: string;
  appraisalPrice: number;
  minimumBidPrice: number;
  depositAmount: number;
  endAt: string;
  detailUrl?: string;
  status: "open" | "closed";
  actualPrice?: number;
  resultDate?: string;
};
```

---

### 15.2 참여 데이터

```ts
type Prediction = {
  id: string;
  email: string;
  nickname?: string;
  predictedPrice: number;
  createdAt: string;
};
```

---

diff = Math.abs(predictedPrice - actualPrice)
